2. Bloqueadores Críticos (Must Fix)

BLOQUEADOR 1 — Zero sistema de pagamentos

  • A landing page anuncia 3 planos (Grátis €0, Pro €29/mês, Enterprise sob consulta)
  • Ambos os botões "Começar" levam diretamente ao /auth?tab=signup
  • Não existe Stripe, PayPal, ou qualquer integração de pagamento
  • Não existe tabela de subscriptions na base de dados
  • Não existe enforcement de limites (o plano "Grátis 50 faturas/mês" nunca é verificado)
  • Resultado: todos os utilizadores têm acesso total gratuito ilimitado

BLOQUEADOR 2 — Formulário de contacto é falso

  • Contact.tsx handleSubmit() faz await new Promise(resolve => setTimeout(resolve, 1000))
  • Mostra "Mensagem Enviada!" ao utilizador — mas nenhuma mensagem é transmitida a ninguém
  • Não existe SendGrid, Resend, SMTP, ou qualquer serviço de email integrado
  • Emails de welcome não existem — o utilizador regista-se e não recebe nada
  • Password reset usa os templates genéricos do Supabase (email noreply@mail.app.supabase.io)
  • Um cliente com um problema de suporte não consegue contactar-nos

3. Problemas de Alta Prioridade

Landing Page — Testemunhos falsos

"Dra. Mariana Lopes (OCC nº 98XXX)", "Ricardo Mendes", "Sofia Carvalho" — estas pessoas não existem. O nº OCC está explicitamente mascarado. Isto é uma prática comercial desleal sob o Decreto-Lei 57/2008 português. Remover ou substituir por testemunhos reais.

Multi-tenancy — Funciona apenas para 1 firma

O modelo usa profiles.accountant_id (flat FK). Não existe conceito de "firma" ou "organização". Se duas firmas concorrentes usarem a plataforma, há risco de fuga de informação via admin views. Para SaaS público, é necessário um nível organization_id.

RGPD — Sem eliminação de conta e sem export completo

  • Não existe funcionalidade "Apagar a minha conta" — a Privacy Policy promete este direito
  • Não existe "Exportar todos os meus dados" (data portability)
  • Data de atualização dos Termos/Privacidade: "Dezembro 2024" (desatualizado)
  • Sem entidade legal identificada (nome da empresa, NIF, morada) — obrigatório por lei

Rate Limits de IA — Sem quotas por utilizador

Um utilizador pode fazer upload de 50.000 documentos e consumir todo o budget OpenRouter/Gemini. Não existe tracking de custos de IA por utilizador, nem limites de uso por plano.

6. Arquitetura Técnica

Frontend

React 18 + TypeScript + Vite (SWC). Tailwind CSS 3 + shadcn/ui. React Query. Code-split com React.lazy. PWA. Deploy: Vercel (auto-deploy de main).

Backend

Supabase (PostgreSQL 15 + Edge Functions Deno + Auth + Storage + RLS). pg_cron scheduler. 94 migrations versionadas. Frankfurt.

IA

Google Gemini via OpenRouter. 6 edge functions: extract-invoice-data, classify-invoice, classify-sales-category, extract-withholding, parse-excel-with-ai, parse-credentials-pdf.

AT Connector

Node.js em VPS dedicado. SOAP/mTLS. WS-Security (AES-128-ECB + RSA). Certificados digitais AT. Endpoint: /api/efatura.

Fluxo de Dados Principal

Upload: User → Camera/File → parse-qr (QR) OU extract-invoice-data (IA) → classify-invoice (DP) → invoices table → Validation UI

AT Sync: pg_cron → run_scheduled_at_sync() → sync-queue-manager → at_sync_jobs → process-at-sync-queue → AT Connector VPS → SOAP AT → invoices/sales_invoices tables

Export: invoices table → dpExcelGenerator.ts → Excel DP / saftExporter.ts → SAF-T XML

9. Roadmap para SaaS Multi-Empresa

Se amanhã quiseres ter mais empresas e escalar como SaaS, estes são os passos por ordem de prioridade.

P0 — Sem isto não se pode cobrar (Semana 1-2)

Stripe Billing

Integrar Stripe Checkout + Customer Portal + Webhooks. Tabela subscriptions na BD. Middleware que verifica plano ativo antes de permitir operações. Planos: Free (50 faturas/mês, 1 cliente), Pro (ilimitado, multi-cliente), Enterprise (API access, SLA).

Email Transacional

Configurar Resend ou SendGrid. Substituir Supabase email templates por branded templates (IVAzen logo, cores). Emails: welcome, password reset, deadline reminders, contacto confirmação, recibo de pagamento.

Formulário de Contacto Real

Wire Contact.tsx a uma edge function que envia email via Resend. Alternativa: integrar Crisp/Intercom para chat ao vivo.

P1 — Necessário para confiança do cliente (Semana 2-3)

RGPD Compliance

Adicionar "Apagar minha conta" em Settings (soft delete + 30 dias + hard delete). Adicionar "Exportar todos os meus dados" (JSON/ZIP). Atualizar Termos/Privacidade com nome legal, NIF, data março 2026.

Landing Page Honesta

Substituir testemunhos falsos por citações reais da Adélia / Accounting Advantage (com permissão). Adicionar vídeo demo real (screen recording 2 min). Verificar claims de precisão ("98% QR" → citar fonte ou remover).

Quotas de IA por Plano

Tabela ai_usage (user_id, date, calls, tokens). Free: 50 classificações/mês. Pro: 5.000. Enterprise: ilimitado. Dashboard de uso em Settings.

P2 — Escalar para múltiplas firmas (Semana 3-4)

Multi-tenancy: Organizações

Nova tabela organizations (id, name, nif, billing_email). FK organization_id em profiles. RLS por organização. Admin por firma (não global). Billing por organização.

Conectores de Faturação (Moloni + InvoiceXpress)

OAuth 2.0 flow para Moloni. API Key config para InvoiceXpress. Sync automático de faturas emitidas. Mapeamento campos → invoices/sales_invoices table. Cobertura: ~40% do mercado PT.

P3 — Diferenciação competitiva (Mês 2+)

Conectores adicionais

Jasmin, PHC GO, Sage 50, Vendus, Colibri. Cada conector = ~3-5 dias. Total: 3-4 semanas para os 10.

Admin Dashboard avançado

Revenue dashboard (MRR, churn, LTV). Custos IA por utilizador. Health dashboard operacional. User impersonation para suporte. Announcements system.

Uptime Monitoring

Pingdom/UptimeRobot para frontend, edge functions, VPS connector. Alertas Slack/email quando algo cai.

Marketplace de Integrações

Página pública "Integrações" mostrando todos os software compatíveis. Badge "Compatible with IVAzen" para parceiros. API pública para third-party integrations.
