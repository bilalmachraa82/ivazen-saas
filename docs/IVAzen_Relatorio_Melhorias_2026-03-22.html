3. Melhorias Implementadas — Detalhe

3.1 Configuracao do Regime IVA por Cliente

Antes

Sistema mostrava alertas de IVA mensal E trimestral para todos os clientes. Sem opcao de configurar o regime.

Depois

Novo campo "Cadencia IVA" (mensal/trimestral) em Definicoes > Editar Cliente. Prazos fiscais respeitam o regime configurado.

3.2 Portfolio Readiness Card (Carteira Interactiva)

Antes

Dashboard mostrava badges estaticos sem accao. Contabilista nao conseguia filtrar clientes por estado.

Depois

Badges clicaveis com contagem de clientes. Ao clicar, aparece lista filtrada com nome, NIF e navegacao directa ao Centro Fiscal.

3.3 Performance — Periodos Fiscais via RPC

Antes

Sistema descarregava 166.000 linhas da base de dados so para extrair ~20 periodos unicos. Lento para carteiras grandes.

Depois

Nova funcao RPC get_distinct_fiscal_periods retorna apenas os ~20 periodos distintos. Seguranca INVOKER + validacao whitelist.

3.4 Performance — Contagem de Clientes via RPC

Antes

Dashboard da Adelia descarregava 53.000 linhas de vendas para contar por cliente no browser.

Depois

Nova funcao RPC count_rows_by_client retorna ~242 pares (cliente, contagem) via GROUP BY server-side.

3.5 Exclusao em Massa de Faturas

Antes

So era possivel excluir faturas uma a uma do calculo do IVA.

Depois

Botao "Selecionar" permite escolher multiplas faturas e aplicar "Nao contabilizar" em massa. Faturas permanecem no sistema mas sao excluidas do calculo.

3.6 Filtro de Importacoes Recentes

Antes

Ao importar novas faturas, o sistema misturava todas na validacao. Impossivel encontrar as recem-importadas.

Depois

Novo filtro "Ultimas 24h" e "Ultimos 7 dias" na pagina de validacao. Prioriza faturas acabadas de importar.

3.7 Enriquecimento de Nomes de Fornecedores

Antes

Muitas faturas mostravam apenas o NIF do fornecedor, sem nome, especialmente na reconciliacao AT.

Depois

Sistema de resolucao multi-fonte: consulta directorio de fornecedores, faturas existentes e metricas para resolver NIF em nome. Reconciliacao AT agora mostra nomes.

3.8 Duplicados Mantêm Copia Uploaded

Antes

Ao limpar duplicados, sistema podia manter a versao AT (sem imagem) e apagar a versao uploaded (com imagem).

Depois

Algoritmo de scoring atribui +100 pontos a copias uploaded. A versao com imagem e sempre preservada.

4. Seguranca

Auditoria de Seguranca Independente

Todas as novas funcionalidades foram auditadas por um revisor de seguranca especializado:

  • SECURITY INVOKER — funcoes RPC executam sob a identidade do utilizador, sem bypass de permissoes
  • Whitelist de tabelas — previne injecao SQL em nomes de tabela dinamicos
  • REVOKE PUBLIC — funcoes acessiveis apenas a utilizadores autenticados
  • Filtragem client-side — pesquisa de vendas movida para client-side para prevenir injecao de filtros PostgREST
  • Credenciais E2E — removidas credenciais hardcoded dos testes, agora via variaveis de ambiente

Resultado: 0 vulnerabilidades CRITICAL, 0 HIGH.
